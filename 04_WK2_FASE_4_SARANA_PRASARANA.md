## DELIVERABLES

### D1: INSPEKSI KONDISI FISIK SEMUA RUANG
**Deadline:** Minggu 1 Juli (WEEK 1!)  
**PIC:** WK2 + Kepala Sarpras  
**Status:** ☐

🚨 **JANGAN TUNDA - Harus minggu 1 Juli, tidak boleh minggu 2!**

**Ruang yang di-inspect:**
- [ ] Semua ruang kuliah (besar & kecil)
- [ ] Ruang dosen (office per dosen)
- [ ] Ruang Kaprodi
- [ ] Ruang rapat/meeting room
- [ ] Mushola/tempat ibadah
- [ ] Toilet (pria & wanita - semua lantai)
- [ ] Area parkir (indoor/outdoor)
- [ ] Koridor & tangga
- [ ] Lift (jika ada)
- [ ] Akses disabilitas (ramp, pintu lebar, toilet khusus)

**Per ruang, dokumentasikan:**
1. **Kondisi dinding:** OK / kecil rusak / besar rusak / perlu cat ulang?
2. **Kondisi lantai:** Bersih / ada noda / tile pecah / licin / butuh polish?
3. **Kondisi ceiling:** OK / ada bocor / plafon jatuh / AC drip?
4. **AC/pendingin:** Berfungsi / tidak ada / error / butuh service?
5. **Pencahayaan:** OK / lampu mati / terlalu dim / flickering?
6. **Furniture:** Meja OK / kursi OK / rak rusak / lemari kunci OK?
7. **Keamanan:** Pintu lock OK / kunci rusak / ada jendela pecah?
8. **Akses:** Pintu lancar / tangga aman / ramuan OK?
9. **Kebersihan:** Bersih / ada sampah / debu / kabel berantakan?
10. **Catatan special:** Apa ada perbaikan mendesak?

**Proses:**
1. Buat inspection form (checklist di atas - bisa print atau digital)
2. Tim inspeksi keliling semua ruang (ambil photo/video evidence)
3. Setiap ruang diisi form & tandatangani inspector
4. Compile semua findings dalam report

**Output:**
- Inspection report dengan foto evidence per ruang
- Daftar kerusakan (prioritas: urgent/critical vs maintenance)
- Rekomendasi perbaikan

### D2: IDENTIFIKASI KERUSAKAN & AJUKAN PERBAIKAN DARURAT
**Deadline:** Minggu 1 Juli (SAME DAY as D1)  
**PIC:** Kepala Sarpras (WK2 approval)  
**Status:** ☐

**Segmentasi kerusakan:**

**URGENT (H+1 laporan, harus selesai minggu 1-2):**
- AC tidak berfungsi di ruang kuliah utama (panas!)
- Toilet tidak bisa flush (sanitary issue!)
- Pintu rusak/tidak bisa kunci (keamanan!)
- Kabel exposed (safety hazard!)
- Lantai licin/berbahaya (fall risk!)
- Jendela pecah (keamanan!)

**PENTING (Minggu 2-3, harus selesai sebelum kuliah):**
- Proyektor error/tidak normal
- Lampu dim/mati di beberapa ruang
- Cat mengelupas (tapi tidak urgent)
- Furniture rusak tapi masih usable (ganti minggu depan OK)
- WiFi signal weak di area tertentu

**MAINTENANCE (Bisa after semester starts):**
- Paint touch-up
- Cleaning profesional
- Minor furniture repair
- Upgrade non-critical

**Proses:**
1. Prioritize berdasarkan kategori di atas
2. Buat form "Permintaan Perbaikan Darurat":
   - Lokasi ruang
   - Deskripsi kerusakan & photo
   - Prioritas (urgent/penting/maintenance)
   - Estimasi biaya
   - Timeline diinginkan
3. Submit ke Yayasan/Ketua untuk approval budget
4. Coordinate dengan contractor
5. Schedule perbaikan (target: selesai minggu 2 untuk urgent, minggu 3 untuk penting)
6. Dokumentasi: sebelum perbaikan & sesudah perbaikan (photo)

**Output:**
- Form permintaan perbaikan (filled & approved)
- PO ke contractor
- Schedule perbaikan
- Before/after documentation

- [ ] Proyektor di setiap ruang kuliah (ada? berfungsi?)

**Testing process:**

   - Update Windows/Mac OS (jika ada)
   - Update proyektor firmware (jika available)
   - Install/update driver proyektor

3. **Dokumentasi:**
   - Create spreadsheet: Ruang, Proyektor Model, Status (OK/Error), Action Taken
   - For error devices: list sparepart yang dibutuhkan

**Output:**
- IT/Proyektor status report (per ruang)
- List of items to repair/replace
- Software update records
- Test results (signed off by UPT TI)

### D4: KONEKSI INTERNET & WiFi KAMPUS
**Deadline:** Minggu 2 Juli  
**PIC:** WK2 + UPT TI  
**Status:** ☐

**Testing:**
1. **Internet connection:**
   - Check speed (Mbps download/upload) - compare dengan contracted speed
   - Check latency/ping
   - Uptime status (any downtime? maintenance?)
   - Backup internet (ada? automatic failover?)

2. **WiFi Coverage:**
   - Map signal strength per area:
     - Ruang kuliah utama (strong?)
     - Ruang dosen (accessible?)
     - Area common space (library, cafeteria)
     - Admin area
   - Signal quality (5G atau 2.4G?)
   - Capacity test (berapa device bisa connect simultaneously?)

3. **Keamanan WiFi:**
   - Password protection (sudah ganti? weak?)
   - Encryption (WPA2 atau WPA3?)
   - Guest network (ada? untuk visitor?)

4. **Bandwi capacity:**
   - Peak usage time (jam berapa paling padat?)
   - Current utilization
   - Proyeksi TA ganjil: apakah kapasitas cukup untuk kuliah + admin + student access?
   - Rekomendasi: upgrade bandwidth? atau optimize?

**Kontrak ISP:**
- Verifikasi SLA (Service Level Agreement) - pastikan provider tahu timing support
- Emergency support number (24/7?)
- Payment status (tidak ada tunggakan yang bikin koneksi diputus!)

**Output:**
- Internet/WiFi speed test results
- Coverage heat map (visual)
- Capacity analysis & recommendations
- ISP contract review (summary)

### D5: VERIFIKASI KAPASITAS RUANG KULIAH
**Deadline:** Minggu 2 Juli  
**PIC:** WK2 (koordinasi dengan WK1 untuk jadwal final)  
**Status:** ☐

**Data yang dibutuhkan:**
- Dari WK1 Phase 2: jadwal perkuliahan final (berapa kelas, berapa mahasiswa per kelas)
- Dari Registrasi: berapa mahasiswa total TA ganjil (expected enrollment)

**Proses:**
1. **Audit kapasitas setiap ruang:**
   - Ruang A: kapasitas 40 kursi, ada AC, proyektor OK
   - Ruang B: kapasitas 60 kursi, no AC, proyektor OK
   - Ruang C: kapasitas 30 kursi, ada AC, proyektor error

2. **Match dengan jadwal:**
   - Kelas Algoritma: 45 mahasiswa → butuh ruang minimal 50 kursi
   - Kelas Database: 35 mahasiswa → bisa di ruang 40-kursi
   - dll

3. **Identifikasi problems:**
   - Mismatch class size dengan available room?

4. **Koordinasi dengan WK1:**
   - Jika ada issue, report ASAP
   - Coordinate alternative ruang (jika perlu)
   - Update jadwal jika butuh pindah ruang

**Output:**
