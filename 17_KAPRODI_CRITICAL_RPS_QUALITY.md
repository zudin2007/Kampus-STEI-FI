# CRITICAL GUIDE: RPS QUALITY ASSURANCE

**Owner:** Kaprodi  
**Deadline:** H-1 kuliah  

---

## PURPOSE

RPS (Syllabus) quality directly affects learning outcomes.

**Poor RPS example:**
- CPMK vague ("students will learn economics")
- Assessment doesn't match CPMK (teach theory, test calculation)
- No rubric → grading arbitrary

---

## QUALITY CHECKLIST

Before approve RPS, verify:

- [ ] CPMK (Course Learning Outcome) clearly stated (SMART: specific, measurable)
- [ ] Assessment methods aligned with CPMK
- [ ] Rubric provided (grading criteria explicit)
- [ ] References relevant & recent
- [ ] Schedule realistic (17 weeks, fits timing)
- [ ] Attendance policy stated
- [ ] Learning resources available (textbook, lab, etc)

---

## FEEDBACK CYCLE

- Dosen submits RPS
- Kaprodi reviews using checklist
- If issues → feedback to dosen
- Dosen revises
- Kaprodi approves
